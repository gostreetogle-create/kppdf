# Документация KP PDF

Читай в таком порядке:

1. [README.md](../README.md) — быстрый старт, запуск проекта
2. [architecture.md](./architecture.md) — структура, компоненты, реактивность
3. [business-logic.md](./business-logic.md) — бизнес-логика, сущности, правила расчётов
4. [api.md](./api.md) — REST API, схемы запросов/ответов
5. [ui-kit.md](./ui-kit.md) — UI kit, design tokens, примеры использования
6. [deploy.md](./deploy.md) — деплой на сервер, переменные окружения

---

## Как вести документацию

- При добавлении нового компонента — обновить `architecture.md` (таблица smart/dumb)
- При изменении API — обновить `api.md`
- При изменении расчётов или статусов — обновить `business-logic.md`
- При добавлении UI kit компонента — обновить `ui-kit.md`
- При изменении деплоя — обновить `deploy.md`
