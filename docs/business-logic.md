# Бизнес-логика

## Сущности

### Product — Товар

Элемент каталога. Используется как шаблон при добавлении в КП.

| Поле          | Тип     | Обязательно | Описание                        |
|---------------|---------|-------------|---------------------------------|
| `name`        | string  | ✅           | Название товара/услуги          |
| `description` | string  | —           | Краткое описание                |
| `unit`        | string  | ✅           | Единица измерения               |
| `price`       | number  | ✅           | Цена за единицу, ₽ (>= 0)      |
| `imageUrl`    | string  | —           | URL изображения                 |

Допустимые единицы измерения: `шт.`, `м²`, `м.п.`, `кг`, `т`, `комплект`, `рейс`, `услуга`

---

### KP — Коммерческое предложение

| Поле          | Тип        | Описание                                      |
|---------------|------------|-----------------------------------------------|
| `title`       | string     | Название КП (для внутреннего использования)   |
| `status`      | KpStatus   | Статус (см. ниже)                             |
| `recipient`   | object     | Данные получателя                             |
| `metadata`    | object     | Параметры КП (номер, сроки, условия оплаты)   |
| `items`       | KpItem[]   | Список позиций                                |
| `conditions`  | string[]   | Дополнительные условия (текстовые строки)     |
| `vatPercent`  | number     | Ставка НДС, % (по умолчанию 20)               |

#### Получатель (recipient)

| Поле    | Тип    | Описание          |
|---------|--------|-------------------|
| `name`  | string | Название компании |
| `inn`   | string | ИНН               |
| `email` | string | Email             |
| `phone` | string | Телефон           |

#### Метаданные (metadata)

| Поле                | Тип    | По умолчанию | Описание                    |
|---------------------|--------|--------------|-----------------------------|
| `number`            | string | —            | Номер КП (напр. КП-2024-001)|
| `validityDays`      | number | 10           | Срок действия, дней         |
| `prepaymentPercent` | number | 50           | Предоплата, %               |
| `productionDays`    | number | 15           | Срок изготовления, дней     |

#### Позиция КП (KpItem)

При добавлении товара из каталога в КП — копируются данные товара на момент добавления. Изменение товара в каталоге не влияет на уже созданные КП.

| Поле          | Тип    | Описание                          |
|---------------|--------|-----------------------------------|
| `productId`   | string | ID товара-источника               |
| `name`        | string | Название (скопировано из товара)  |
| `description` | string | Описание                          |
| `unit`        | string | Единица измерения                 |
| `price`       | number | Цена за единицу на момент добавления |
| `qty`         | number | Количество (>= 1)                 |
| `imageUrl`    | string | URL изображения                   |

## Статусы КП

```
draft → sent → accepted
              ↘ rejected
```

| Статус     | Метка       | Цвет   | Описание                              |
|------------|-------------|--------|---------------------------------------|
| `draft`    | Черновик    | серый  | КП создано, не отправлено             |
| `sent`     | Отправлен   | синий  | КП отправлено клиенту                 |
| `accepted` | Принят      | зелёный| Клиент принял КП                      |
| `rejected` | Отклонён    | красный| Клиент отклонил КП                    |

Статус меняется вручную в редакторе КП через select в тулбаре.

## Расчёт итогов

Все расчёты выполняются на фронтенде в `KpBuilderComponent` через `computed()`.
На бэкенде итоги не хранятся — пересчитываются при каждом открытии.

```
subtotal  = Σ (item.price × item.qty)
vatAmount = round(subtotal × vatPercent / 100)
total     = subtotal + vatAmount
```

Пример:
```
Товар А: 25 000 ₽ × 2 шт = 50 000 ₽
Товар Б:  5 000 ₽ × 3 шт = 15 000 ₽
─────────────────────────────────────
Subtotal:              65 000 ₽
НДС 20%:               13 000 ₽
Итого:                 78 000 ₽
```

## Логика редактора КП (KpBuilderComponent)

### Добавление товара
- Если товар уже есть в составе КП — увеличивает `qty` на 1
- Если нет — добавляет новую позицию с `qty = 1`
- Данные копируются из каталога в момент добавления (снимок)

### Изменение количества
- Минимальное значение: 1
- Изменение через `<input type="number">` в сайдбаре

### Удаление позиции
- Удаляет позицию из `items` по `productId`

### Сохранение
- `PUT /api/kp/:id` — сохраняет весь объект КП целиком
- Автосохранения нет — только по кнопке "Сохранить"

### Печать / PDF
- `window.print()` — браузерная печать
- Элементы с классом `.no-print` скрываются через `@media print`
- Документ оптимизирован под формат A4

## Многостраничность документа

`KpDocumentComponent` автоматически разбивает товары на страницы:

- `itemsPerPage` — количество позиций на страницу (по умолчанию 10)
- Страница 1: фон `kp-1str.png` + шапка с получателем и метаданными
- Страницы 2+: фон `kp-2str.png`, без шапки, с отступом `margin-top: 20mm`
- Последняя страница: итоги (`KpTableComponent`) + условия
- Номера страниц показываются если страниц > 1

---

## Каталог товаров

- Товары хранятся в MongoDB, независимо от КП
- Поиск в реальном времени по `name` и `description` (computed signal)
- Два вида отображения: сетка карточек и таблица (переключатель в тулбаре)
- Удаление товара из каталога не удаляет его из уже созданных КП (данные скопированы)

## Нумерация КП

При создании нового КП номер генерируется автоматически на фронтенде:

```typescript
number: `КП-${Date.now()}`
```
