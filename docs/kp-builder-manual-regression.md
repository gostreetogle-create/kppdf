# KP Builder Manual Regression

## Цель
Подтвердить поведение после UX/logic обновлений в создании КП.

## Сценарии

1. **Получатель**
   - Открыть `/kp/:id`.
   - Проверить: видны только базовые поля.
   - Нажать `Показать реквизиты` и убедиться, что раскрываются доп. поля.
   - Ввести некорректные ИНН/КПП/Email и проверить inline-предупреждения.

2. **Каталог**
   - Включить `Фокус: каталог`.
   - Проверить, что секции `Состав`/`Условия` скрываются.
   - Проверить поиск по части названия и фильтр по категории.

3. **Состав КП**
   - Добавить товар.
   - Изменить количество кнопками `-`/`+`.
   - Удалить товар и нажать `Вернуть удалённый товар`.
   - Проверить пересчет итогов.

4. **Условия**
   - Добавить условие из шаблона.
   - Добавить вручную.
   - Переставить условия `↑`/`↓`.
   - Проверить, что дубликаты не добавляются.

5. **Роли**
   - Войти под `manager`.
   - Проверить подсказку ограничений в тулбаре.
   - Убедиться, что доступны только разрешенные переходы статуса.

6. **Печать**
   - Нажать `Печать / PDF`.
   - Проверить, что панели (`no-print`) скрыты.
   - Проверить корректность A4 страниц и переносов.

7. **Медиа**
   - Визуально проверить отображение фото товаров в документе.
   - На странице товаров нажать `Отчёт: фото отсутствуют` и проверить JSON.
