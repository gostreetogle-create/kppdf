## Архитектурный ландшафт
**Стек:** Angular (Frontend) + Node.js/Express (Backend) + MongoDB + Puppeteer.

**Ключевая особенность:** 
Генерация PDF выполняется через Puppeteer. Для оптимизации используется **Browser Pooling** через `PuppeteerService`. 
- **ЗАПРЕЩЕНО:** Вызывать `puppeteer.launch()` внутри сервисов.
- **РАЗРЕШЕНО:** Использовать `puppeteerService.getBrowser()`. Это возвращает существующий экземпляр браузера, что экономит ~2-3 секунды на каждом запросе.
- **ОБЯЗАТЕЛЬНО:** Всегда закрывать страницу через `page.close()` в блоке `finally`, но НЕ закрывать сам браузер.

## Карта проекта

### Backend (`/backend/src`)
- **`services/puppeteer.service.ts`**: Singleton для управления экземпляром Puppeteer.
- **`services/pdf-generator.service.ts`**: Основной сервис генерации КП на основе рендеринга Angular-страниц.
- **`services/product-passport-pdf.service.ts`**: Сервис генерации техпаспортов на основе HTML-шаблонов.
- **`services/kp-pdf.service.ts`**: Легковесный сервис генерации КП через прямой HTML (fallback).
- **`controllers/kp.controller.ts`**: Обработка HTTP-запросов на экспорт и превью.
- **`routes/kp.routes.ts`**: Определение эндпоинтов (разделение на `/export` и `/preview`).

### Frontend (`/frontend/src/app`)
- **`features/kp/kp-builder/`**: Основной конструктор коммерческих предложений.
- **`features/products/components/product-spec-editor/`**: Редактор спецификаций и техпаспортов.
- **`core/services/api.service.ts`**: Точка входа для всех API-запросов, включая ссылки на превью.

## Чек-лист проверки бизнес-логики PDF
При работе с генерацией PDF всегда проверяйте:
- [ ] **Ресурс-менеджмент**: Страница создана через `browser.newPage()` и закрыта через `page.close()` в `finally`.
- [ ] **Шрифты и стили**: Для PDF-экспорта используется CSS-селектор `[data-pdf-export="true"]` или медиа-запрос `@media print`.
- [ ] **Wait Logic**: Использовать `networkidle0` при загрузке и проверять `img.complete` для всех изображений перед вызовом `page.pdf()`.
- [ ] **Content-Disposition**: 
    - `attachment` для скачивания (эндпоинты `/export`).
    - `inline` для просмотра в браузере (эндпоинты `/preview`).
- [ ] **Именование**: Файлы должны иметь безопасные имена (без спецсимволов), сформированные через регулярку `/[^\w.-]+/g`.

## Инструкции по экономии токенов
- **НЕ читай целиком**:
    - `node_modules/`, `dist/`, `.angular/`.
    - Тяжелые JSON-файлы документации или логов.
    - Файлы стилей (`.scss`), если задача не касается верстки PDF.
- **Где искать логику**:
    - Если нужно понять, как данные превращаются в PDF: начни с `kp.controller.ts` -> `pdf-generator.service.ts`.
    - Если нужно изменить UI кнопок экспорта: `kp-builder.component.html`.

## Масштабируемость и Будущее
### Добавление новых документов
Чтобы добавить новый тип документа (например, Счет или Акт):
1. Создайте новый сервис в `backend/src/services/`.
2. Используйте `puppeteerService.getBrowser()` для получения страницы.
3. Реализуйте метод `generateXxxPdf`.
4. Добавьте соответствующие эндпоинты в контроллер с поддержкой `inline/attachment`.

### Рост нагрузки
При значительном росте количества запросов на PDF:
- Рассмотреть вынос `PuppeteerService` в отдельный микросервис или использование Worker Threads.
- Внедрить очередь задач (BullMQ / Redis) для обработки тяжелых PDF в фоновом режиме.
